You can choose between using CSS variables or Tailwind CSS utility classes for theming.

## Utility classes

<ComponentCode
  example={true}
  language="tsx"
  preview={`<div className="bg-zinc-950 dark:bg-white" />`}
/>

To use utility classes for theming set `tailwind.cssVariables` to `false` in your `components.json` file.

<ComponentCode
  example={true}
  language="json"
  preview={`{
  "tailwind": {
    "config": "tailwind.config.js",
    "css": "app/globals.css",
    "cssVariables": false
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils"
  }
}`}
/>

## CSS Variables

<ComponentCode
  example={true}
  language="tsx"
  preview={`<div className="bg-background text-foreground" />`}
/>

To use CSS variables for theming set `tailwind.cssVariables` to `true` in your `components.json` file.

<ComponentCode
  example={true}
  language="json"
  preview={`{
  "tailwind": {
    "config": "tailwind.config.js",
    "css": "app/globals.css",
    "cssVariables": true
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils"
  }
}`}
/>

### Convention

We use a simple `background` and `foreground` convention for colors. The `background` variable is used for the background color of the component and the `foreground` variable is used for the text color.

<Callout className="mt-4">

The `background` suffix is omitted when the variable is used for the background color of the component.

</Callout>

Given the following CSS variables:

<ComponentCode
  example={true}
  language="css"
  preview={`--primary: 222.2 47.4% 11.2%;
--primary-foreground: 210 40% 98%;`}
/>

The `background` color of the following component will be `hsl(var(--primary))` and the `foreground` color will be `hsl(var(--primary-foreground))`.

<ComponentCode
  example={true}
  language="tsx"
  preview={`<div className="bg-primary text-primary-foreground">Hello</div>`}
/>

<Callout>

**CSS variables must be defined without color space function**. See the [Tailwind CSS documentation](https://tailwindcss.com/docs/customizing-colors#using-css-variables) for more information.

</Callout>

### Adding new colors

To add new colors, you need to add them to your CSS file and to your `tailwind.config.js` file.

<ComponentCode
  example={true}
  language="css"
  preview={`:root {
  --warning: 38 92% 50%;
  --warning-foreground: 48 96% 89%;
}`}
/>

<ComponentCode
  example={true}
  language="css"
  preview={`.dark {
  --warning: 48 96% 89%;
  --warning-foreground: 38 92% 50%;
}`}
/>

<ComponentCode
  example={true}
  language="js"
  preview={`module.exports = {
  theme: {
    extend: {
      colors: {
        warning: "hsl(var(--warning))",
        "warning-foreground": "hsl(var(--warning-foreground))",
      },
    },
  },
}`}
/>

You can now use the `warning` utility class in your components.

<ComponentCode
  example={true}
  language="tsx"
  preview={`<div className="bg-warning text-warning-foreground" />`}
/>
